mcxsic commented on Feb 24, 2019

Even though it looks like an issue with Expo, I've found a temporary solution:

Solution

  • Keep using Expo SDK 32.
  • Go Google Developer Console and create an Android Oauth2 Client ID and an iOS Oauth2 Client ID, as stated on the SDK 31 documentation for the Google sign in (make sure that the package name is host.exp.exponent).
  • Use the Android Client ID and iOS Client ID as it follows:
import { Platform } from 'react-native';
export const isAndroid = () => Platform.OS === 'android';

const result = await Google.logInAsync({
    clientId: isAndroid() ? '<YOUR_ANDROID_CLIENT_ID>' : '<YOUR_IOS_CLIENT_ID>',
    scopes: ['profile', 'email']
});

Explanation
The main problem that I've seen is that even though on the new documentation the following is stated:

const clientId = '<YOUR_WEB_CLIENT_ID>';

The truth is that internally the request is being called from the Expo app, and you can see that when you expand the request details on the error page:

as you can see, it adds the package name host.exp.exponent.

Anyway, this will only work while you're in development or in the Expo app. If you create a standalone app, you need to use Google Sign-In and that's a complete different battle.

Also, it would be great if the documentation for SDK 32 it's clearer about how to create the Client ID, since it was clearer on previous versions. In addition, @types/expo still has the old structure ( this is what Google.LogInConfig looks in version ^32.0.3):

export namespace Google {
    interface LogInConfig {
        androidClientId?: string;
        androidStandaloneAppClientId?: string;
        iosClientId?: string;
        iosStandaloneAppClientId?: string;
        webClientId?: string;
        behavior?: 'system' | 'web';
        scopes?: string[];
    }
}

(PS: also weird things are happening with Firebase if you create an app inside your Firebase Project; using React Native + Login + Database these days can be nerve-wracking...)

enekesabel commented on May 10, 2019 •
edited

For anyone struggling with this problem, I managed to solve it using AuthSession, and an extra request. This way you can use your web client credentials to authenticate.

import {AuthSession} from "expo";

// both can be found under https://console.developers.google.com/apis/credentials
const clientId = '<YOUR_WEB_CLIENT_ID>';
const clientSecret = '<YOUR_WEB_CLIENT_SECRET>';

const doAuth = async ()=>{
  const redirectUrl = AuthSession.getRedirectUrl();
  const result = await AuthSession.startAsync({
    authUrl:
      'https://accounts.google.com/o/oauth2/v2/auth?'
      + `&client_id=${clientId}`
      + `&redirect_uri=${encodeURIComponent(redirectUrl)}`
      + '&response_type=code'
      + '&scope=profile',
  });

  const response = await fetch('https://www.googleapis.com/oauth2/v4/token', {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      code: result.params.code,
      client_id: clientId,
      client_secret: clientSecret,
      redirect_uri: redirectUrl,
      grant_type: 'authorization_code',
    }),
  });

  const data = await response.json();
  /*
  "data" contains access_token and id_token needed for firebase credential login
  {
    "access_token": "access_token",
    "expires_in": 3599,
    "id_token": "id_token",
    "scope": "https://www.googleapis.com/auth/userinfo.profile",
    "token_type": "Bearer",
  }
  */
};

cruzach commented on Oct 16, 2019

Hey everyone, closing this as the docs have been updated (for quite a while, looks like this issue slipped through the cracks!)

Now, you need to supply different clientIds for iOS/Android in the Expo client, and standalone iOS/Android:

const { type, accessToken, user } = await Google.logInAsync({
  iosClientId: `<YOUR_IOS_CLIENT_ID_FOR_EXPO>`,
  androidClientId: `<YOUR_ANDROID_CLIENT_ID_FOR_EXPO>`,
  iosStandaloneAppClientId: `<YOUR_IOS_CLIENT_ID>`,
  androidStandaloneAppClientId: `<YOUR_ANDROID_CLIENT_ID>`,
});
